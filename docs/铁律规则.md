# 🔒 铁律规则（100%确定，永不改变）

这些规则经过测试验证，任何情况下都必须遵守。

---

## 零、数据源优先级（最重要）

**默认使用UIA（覆盖90%场景）：**
- `latest.json` + `latest.png`

**仅当UIA失败时才用Vision：**
- 失败条件：`control_count < 5` 或 找不到目标元素
- 切换到：`latest_vision.json`

⚠️ 不要一开始就读取所有文件，先用UIA，不够再切Vision

---

## 一、坐标获取规则

### ✅ 必须做
- **从JSON获取坐标**：所有点击坐标必须从 `latest.json` 的 `index.by_name` 或 `controls` 中获取
- **使用grep_search查找**：`grep_search("目标名称", "v2_simplified/output/latest.json", MatchPerLine=True)`
- **或直接读取索引**：`read_file("v2_simplified/output/latest.json")` 然后读取 `index.by_name.目标名称`

### ❌ 禁止做
- 不要"估计"或"猜测"坐标
- 不要使用MCP工具（如mcp3_click_screen）的坐标，坐标系不一致
- 不要凭记忆使用上次的坐标（界面可能已变化）

---

## 二、中文输入规则

### ✅ 必须做
- **使用剪贴板方式**：
  1. `pyperclip.copy('中文内容')`
  2. `pyautogui.hotkey('ctrl', 'v')`

### ❌ 禁止做
- 不要使用 `pyautogui.typewrite()` 输入中文（不支持）
- 不要使用 `pyautogui.write()` 输入中文

---

## 三、命令格式规则

### ✅ 必须做
- **单行命令格式**：所有Python代码写成一行，语句用分号分隔
- 示例：`python -c "import pyautogui; pyautogui.click(695, 1012)"`

### ❌ 禁止做
- 不要写多行命令（PowerShell会截断）
- 不要在命令中换行

---

## 四、操作验证规则（最重要）

### ⚠️ 核心要求
**每次操作后必须查看截图** `read_file v2_simplified/output/latest.png`

### ✅ 必须做
- **查看截图**：每次点击/输入后，必须 `read_file latest.png` 查看实际效果
- **对比变化**：根据截图判断操作是否成功
- **决定下一步**：基于截图内容决定继续还是换方法

### ❌ 禁止做
- 不要"执行完就算完"，必须查看截图验证
- 不要跳过截图验证步骤

---

## 五、MCP工具使用规则

### ✅ 可以使用
- `read_file` - 读取截图和JSON
- `grep_search` - 搜索JSON中的控件坐标
- `run_command` - 执行pyautogui命令
- `command_status` - 检查命令执行状态

### ❌ 禁止使用
- `mcp3_click_screen` - 坐标系不一致，100%失败
- `mcp0_click` - 坐标系不一致
- 其他MCP点击工具 - 除非明确测试过有效

---

## 六、快速参考

### 标准点击流程
```
1. read_file("latest.json") → 获取坐标 [x, y]
2. run_command: python -c "import pyautogui; pyautogui.click(x, y)"
3. read_file("latest.png") → 查看截图验证效果 ← 必须执行
4. 根据截图决定下一步
```

### 标准输入流程
```
1. 点击输入框（按上述流程）
2. run_command: python -c "import pyperclip; pyperclip.copy('内容'); import pyautogui; pyautogui.hotkey('ctrl', 'v')"
3. 按需按Enter
```

### 启动软件流程
```
1. run_command: python -c "import pyautogui; pyautogui.hotkey('win', 's')"
2. 剪贴板输入软件名称
3. run_command: python -c "import pyautogui; pyautogui.press('enter')"
4. 验证软件是否启动
```
