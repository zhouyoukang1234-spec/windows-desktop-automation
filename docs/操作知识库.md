# 📚 操作知识库

按软件和操作类型分类的操作经验。

---

## 一、通用操作模板

### 1.1 启动软件

**推荐方法：Win+S搜索**
```
步骤：
1. pyautogui.hotkey('win', 's')
2. pyperclip.copy('软件名') + pyautogui.hotkey('ctrl', 'v')
3. pyautogui.press('enter')
4. 验证软件是否启动
```

**注意事项**：
- 如果搜索没结果，可能软件名不对

### 1.2 窗口激活

**方法：通过句柄激活**
```
步骤：
1. 使用win32gui.EnumWindows找到目标窗口句柄
2. win32gui.ShowWindow(hwnd, SW_RESTORE)
3. win32gui.SetForegroundWindow(hwnd)
```

**注意事项**：
- 最小化的窗口需要先Restore
- 窗口标题匹配用模糊匹配

### 1.3 菜单操作

**标准流程**：
```
步骤：
1. 点击菜单标题
2. 从新的JSON获取子菜单项坐标
3. 点击子菜单项
```

**注意事项**：
- 菜单展开前子项不可见
- 多级菜单逐级操作

### 1.4 对话框处理

**标准流程**：
```
步骤：
1. 从JSON查找OK/确定/取消按钮
2. 点击对应按钮
```

**常见按钮名称**：
- 确定类：OK, 确定, 确认, Yes, 是
- 取消类：Cancel, 取消, No, 否
- 关闭类：Close, 关闭, X

## 二、软件专用知识

### 2.1 豆包（Web应用）

**类型**：浏览器内Web应用

**特点**：
- 输入框是contenteditable div，不是标准Edit控件
- 控件识别需要深度遍历Document
- 修复后可识别105+控件

**发送消息流程**：
```
1. grep查找包含"发消息"或"输入"的控件
2. 点击输入框坐标
3. 剪贴板输入内容
4. pyautogui.press('enter')
```

**注意事项**：
- 必须用剪贴板输入中文

### 2.2 Home Assistant（PWA应用）

**类型**：网页转桌面应用

**特点**：
- 数据实时更新
- 开关状态可能随时变化
- 控件名称包含设备名和状态

**开关设备流程**：
```
1. grep查找设备名称（如"四号开关"）
2. 点击开关坐标
3. 验证状态变化（开关名称会变化，如"Toggle xxx off"）
```

**注意事项**：
- 开关控件名称会随状态变化
- 注意区分"打开"和"关闭"状态的坐标

### 2.3 FreeCAD（桌面CAD软件）

**类型**：专业桌面应用

**特点**：
- 启动较慢
- 可能有恢复对话框
- 工具栏密集，菜单层级深

**启动流程**：
```
1. Win+S搜索FreeCAD
2. 如有恢复对话框，点击Cancel
3. 可以开始操作
```

**创建草图流程**：
```
1. 点击"草图"菜单
2. 点击"创建草图"
3. 选择平面（XY-平面）
4. 点击OK
```

**注意事项**：
- 操作顺序严格
- 工具栏位置固定，可缓存坐标

### 2.4 Chrome/Edge（浏览器）

**特点**：
- 网页内容动态变化
- 需要深度遍历Document
- 控件结构复杂

**导航流程**：
```
1. 点击地址栏（通常在顶部）
2. Ctrl+A全选
3. 剪贴板输入URL
4. Enter
5. 等待页面加载
```

## 三、失败恢复策略

### 3.1 点击没反应

**可能原因**：
- 坐标不准确
- 控件被遮挡
- 窗口不在前台

**恢复步骤**：
1. 重新读取最新JSON
2. 重新获取坐标
3. 确认窗口在前台
4. 重试点击

### 3.2 输入内容没出现

**可能原因**：
- 输入框未获得焦点
- 剪贴板内容被清空
- 输入法问题

**恢复步骤**：
1. 再次点击输入框
2. 重新执行剪贴板输入

### 3.3 菜单没展开

**可能原因**：
- 点击位置偏移
- 菜单需要右键点击
- 窗口未激活

**恢复步骤**：
1. 确认窗口激活
2. 重新获取菜单坐标
3. 尝试再次点击

### 3.4 软件没启动

**可能原因**：
- 软件名拼写错误
- 搜索结果未选中
- 软件需要管理员权限

**恢复步骤**：
1. 按Escape关闭搜索
2. 尝试其他软件名变体
3. 或手动指定完整路径

## 四、性能优化建议

### 4.1 减少等待时间
- 熟悉的操作可减少等待
- 但首次操作应保守等待

### 4.2 批量操作
- 同一窗口的多个操作可连续执行
- 窗口切换时需要重新获取坐标

### 4.3 坐标缓存
- 固定UI（如工具栏）坐标可临时缓存
- 动态内容必须每次重新获取
